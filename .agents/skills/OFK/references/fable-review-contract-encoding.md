# Fable Review — Data Contract / Feature Encoding (V29)

**Date Recorded**: 2026-07-20
**Related Files**: [contract.py](file:///c:/REPO/Antigravity/AIPoker/versions/v29/core/contract.py), [train.py](file:///c:/REPO/Antigravity/AIPoker/versions/v29/self_play/train.py), [simulator.py](file:///c:/REPO/Antigravity/AIPoker/versions/v29/self_play/simulator.py), [manifest.py](file:///c:/REPO/Antigravity/AIPoker/shared/manifest.py), [table_state.py](file:///c:/REPO/Antigravity/AIPoker/core/table_state.py), [PHPHelp.py](file:///c:/REPO/Antigravity/AIPoker/PHPHelp.py)

## Context

Contract/encoding area report from the 2026-07-20 four-way V29 audit (see
`fable-review-consolidated.md` for the cross-area synthesis and verification status). Scope:
train/serve encoding mismatches, scaling, information loss, sentinel collisions, versioning.

## Ranked findings

**1. HIGH — Live action-history channel is permanently empty (all-PAD) while training always fed real tokens.**
`PHPHelp.py:1323-1331` (the only live `make_decision` call) never passes `action_history_raw`; `core/decision.py:491` forwards `None`; `versions/v29/core/contract.py:308-312` then leaves `act_ints` all zeros. Training and self-play rollouts always populate it (`versions/v29/self_play/train.py:353-358,470` — tokens 7/3/6; `simulator.py:1546,1558,1583` — `hero_actions_histories` threaded into every query).
Failure scenario: on any multi-decision hand the transformer's action-embedding input is a token stream it never saw during training (states present, actions PAD). Any learned dependence on "I raised the flop" silently degrades at the real table, and nothing flags it — the tensors are shape-valid.

**2. HIGH — Rollout seat activity is assigned by slot index, not by who actually folded.**
`versions/v29/self_play/simulator.py:597`: `is_active = (idx < num_opponents)` marks the first N ascending-seat slots active. The gradient tensors use the true per-seat mask (`simulator.py:1476-1479` → `train.py:434`), and live uses real `is_active` flags.
Failure scenario: seat_1 folds UTG, seat_5 is still in → the acting policy sees seat_1 "active" (with its HUD colors/position) and seat_5 "inactive", and — because `committed`/`raised_*` lookups are gated on this same wrong flag (`simulator.py:635-641`) — the truly-active seat's committed chips and raise flags are zeroed. Every self-play trajectory is generated by a policy reading features that disagree with the encoding it is gradient-trained and live-served on.

**3. HIGH — The V27 [OPP-7] seat re-mapping produces seat keys the contract cannot read, silently defeating the fix.**
`simulator.py:593-596` keys the 5 opponent slots by absolute seat id (`seat_0`…, skipping `seat_{actor}`), but `contract.py:195,243` only ever reads `seat_1`–`seat_5`.
Failure scenario: for a lagged-self/NN opponent at seat 3, its `seats` dict is {seat_0, seat_1, seat_2, seat_4, seat_5} — the contract finds `seat_3` missing (phantom absent-seat defaults) and never reads `seat_0`, so the real hero is invisible to every non-hero NN query. The stated purpose of OPP-7 ("the real hero appearing whenever actor_seat > 0") does not survive tensor encoding; its active-mask sum is also off by one.

**4. HIGH — A failed weight load is swallowed and then served live.**
`core/models/v29_engine.py:66-68` catches the load exception, prints, sets `loaded=False`; `core/decision.py:442-444` checks only that the engine object exists, never `.loaded`.
Failure scenario: `expert_main.pth` missing/corrupt at app start → live play samples actions from randomly initialized weights at a real-money table, with only a console line — the fail-loud checkpoint machinery was explicitly built to prevent exactly this.

**5. MED-HIGH — `contract_version` is never actually validated; only `context_dim` is.**
`shared/manifest.py:82-88` compares `context_dim` and merely *prints* `contract_version` in the error text.
Failure scenario: a V19 checkpoint (35-dim, contract v3) loads cleanly into the V20 engine (35-dim, contract v4 — the 4× money rescale) — same width, different scale, silent garbage: exactly the class of bug the module docstring claims is impossible. V29 (54/8) is protected only by accidentally having a unique width today; the next same-width contract bump re-arms this.

**6. MED-HIGH — Live equity is computed with a different estimator than the trained-on feature.**
Live: `PHPHelp.py:1160-1165` calls range-aware equity with **no `front_colors`** (250 sims) and falls back to vs-random when HUD colors are absent (`PHPHelp.py:1212-1218`). Training: `simulator.py:1423-1434` splits front (already committed — no fold-roll) vs after opponents, 150 sims + noise.
Failure scenario: hero faces a bet from an opponent who has already committed — live gives that opponent a VPIP fold-roll the training distribution never applied, so ctx[3] (and `equity_edge` derived from it, `contract.py:273`) is systematically shifted in exactly the "facing aggression" spots where the equity feature matters most.

**7. MED — Rollout opponent stacks are a placeholder (= hero's own stack).**
`simulator.py:645`: `stack=hero_stack if is_active else 0.0`. Gradient tensors use real per-seat stacks (`train.py:436`, from `simulator.py:1479`); live uses real OCR stacks.
Failure scenario: same rollout-vs-gradient split as #2 — the acting policy in self-play never sees a real covering/short opponent stack, so the state distribution the targets are estimated on is generated blind to stack asymmetry, while live inference feeds real (potentially very different) values.

**8. MED — Live unknown-opponent HUD default encodes a super-nit; training's unknown default encodes an average player.**
`core/table_state.py:340-341` defaults `vpip_color/agg_color` to `"Blue"` → 0.10/0.18 via `contract.py:50-51`; training's absent-profile default is 0.30/0.46 (`train.py:401,429`; contract absent-seat default Yellow/Green at `contract.py:254`).
Failure scenario: HUD badge not yet OCR'd (new player sits, badge obscured) → the model reads the villain as the tightest possible archetype and over-bluffs / over-folds-to-aggression against a completely unknown opponent.

**9. MED — Live `committed` features exclude blinds and inherit an inference-not-ground-truth action classifier.**
Live committed = first-observed stack minus current (`table_state.py:222-224,311-312,343`); the first observation almost always happens after blinds are posted, while the sim counts blinds in `committed` from deal (`simulator.py:1331-1335`). The raise/call classifier (`table_state.py:242-252`) works on per-tick stack diffs — a missed frame merging two actions, or an OCR under-read (monotonic `min` at line 172 makes it permanent), flips call→"raise".
Failure scenario: SB/BB seats are short ~0.5-1bb in `opp_committed_bb` every hand; one glitched frame sets `raised_this_hand`/increments `pot_type` for a passive caller — the exact per-seat aggression signal V29 was built to add arrives with a noise floor training never had (training reads ground-truth events).

**10. MED — Partial board reads alias to "River".**
`contract.py:186-190`: `board_len` not in {0,3,4} → `street_level=3.0`; `table_state.py:304` emits street "Unknown" for 1-2 cards but the contract only counts cards.
Failure scenario: a 2-card mid-deal/partial OCR frame encodes as river with a 2-card board tensor — a state with zero training support — on the same tick a decision can be requested.

**11. MED — Three parallel encoders remain, and they already disagree.**
`contract.py::to_tensors` (rollout + live), `train.py::vectorize_hand_samples:410-467` (gradient). Money scaling was unified after the V20 incident (`contract.py:28-48`), but masks, seat mapping, HUD lookup, and stack sourcing are still hand-duplicated — findings #2/#3/#7 are precisely drift in the unshared parts.
Failure scenario: the next appended feature gets one gated lookup pattern in one builder and another in the other; nothing structural detects it (model_verify runs against the simulator's own encoder, so it shares the distortion).

**12. MED — Short-handed live tables are outside training support, and position arithmetic assumes gap-free seating.**
The sim always deals 6 players; live DoN endgames are 3-5 handed. Absent seats encode as (mask 0, stack 0, pos −1, Yellow/Green) — training's inactive seats always carried real stacks/colors of dealt players. Position formulas `(0 - dealer_idx) % 6` (`table_state.py:191,411`) and `(j+1+hero_position) % 6` (`contract.py:257`) are seat arithmetic; with empty seats between button and hero, blinds skip seats and every encoded position is wrong.
Failure scenario: 4-handed bubble (the highest-stakes phase of DoN), button at seat 4, seat 5 empty → hero encoded as BB while actually SB; every opponent's position feature is similarly shifted.

**13. LOW-MED — Saturation, aliasing, and sentinel collisions.**
- `contract.py:43-48`: stack/call clamp at 100bb — 100bb and 250bb tables encode identically (accepted V22 tradeoff, but live cash tables exceed it routinely).
- `contract.py:50-51`: VPIP quantization — a 36% and an 80% VPIP both encode as 0.45 (Red); all reads are 4-valued.
- `contract.py:227`: `active_mask/10.0` — max 5 opponents, so the feature lives in [0, 0.5]; half the range is dead.
- `contract.py:76`: `equity_edge` is unnormalized (0-6), an order-of-magnitude scale outlier vs every other ~[0,1] feature.
- `contract.py:274` / `core/board_state.py:58`: `hand_strength` default 0.5 collides with genuinely mediocre hands (~K7o); a failed live computation (`PHPHelp.py:1208-1209`) is indistinguishable from a real 50% hand.
- `PHPHelp.py:1300`: call button unavailable → hardcoded `call_amount = 100.0` chips fed into pot_odds/scaled_call as if a real amount (plus `2.0` fallbacks at 1292-1294).
- `contract.py:294`: `pot_type` is documented as limped/single-raised/3-bet+ but counts raise events across *all* streets — a flop bet + turn bet reaches bucket 2 with no 3-bet (consistent train/live, but the semantic is misleading and caps 3-bet vs 5-bet into one value).

**14. LOW — Information the pipeline has but the snapshot drops.**
- Raise *sizes* in history: `train.py:354-358` collapses every raise to one token ('r'), and the size-blindness of history tokens is the still-open [OPP-3]; the sim has `raise_size` at every event and drops it.
- Per-street contributions: `street_committed` exists (`simulator.py:1378`) but only whole-hand `committed` is encoded — donk-bet vs check-raise lines with equal totals alias.
- Current aggressor: `last_raiser` is tracked (`simulator.py:1403`) but not encoded; `raised_this_street` says who has raised, not who made the *live* bet hero currently faces when two seats raised.
- Hero's own aggression: live tracks `raised_this_hand['Hero']` (`table_state.py:249`) but no hero-side raise feature exists.

**15. LOW — Hygiene.**
- `contract.py:11-22` `card_to_int` is case-sensitive while `train.py:60-68`'s copy normalizes case — a lowercase-rank card silently becomes PAD (52) in one encoder only; malformed OCR cards alias to "no card".
- `contract.py:9` `VOCAB` is dead code in this file (tokens arrive as ints from callers).
- `core/decision.py:446-459`: `hand_history_buffer` appends on *every* `make_decision` call — a re-triggered evaluation within one turn inserts duplicate states (training has exactly one state per decision); and two consecutive preflop-only hands with identical hole cards never reset the buffer.
- Stale docs: `core/models/v29_engine.py:12-19` and `versions/v29/core/manifest.py:37-47` still state the 10 OPP-2 live features are "NOT yet functional in LIVE serving / fed as constant 0", but `core/table_state.py` now populates them (and `decision.py:184-201` says so) — a future session trusting the manifest could re-implement or "fix" wiring that already exists.

## What's actually solid

- **Append-only index stability** is genuinely enforced across V20→V29 (indices 0-43 byte-stable, new features strictly appended), and the per-version bridge separation in `decision.py:375-392` correctly prevents cross-scale feeding.
- **The V20 lesson was institutionalized**: money scaling goes through single shared helpers (`scaled_stack_bb`/`scaled_pot_bb`/`scaled_call_bb`, `contract.py:54-65`) imported by both encoders, with the ceiling/scale rationale documented at the constants.
- **Color↔midpoint consistency is exact**: `VPIP_MAP`/`AGG_MAP` (`contract.py:50-51`), `map_*_to_midpoint` (`train.py:70-80`), and the sim's threshold bands (`simulator.py:623-629`) all agree to the digit.
- **Checkpoints are self-describing and width-checked fail-loud** (`shared/manifest.py`), the free-check fold mask and temperature sampling are mirrored train↔serve (`decision.py:549-559`), left-padding/sequence truncation is identical in both encoders, and `equity_edge` is derived inside each encoder from already-present inputs rather than threaded (one less drift surface).
- The V29 live wiring itself (per-seat raise flags, committed, pot_type, all-in stack handling in `table_state.py`) is a real, thoughtful upgrade — the critiques above are about its noise floor and defaults, not its existence.
